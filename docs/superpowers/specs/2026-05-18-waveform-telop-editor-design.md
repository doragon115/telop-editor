# 波形テロップエディター 設計仕様

**日付:** 2026-05-18  
**対象ファイル:** `subtitle-editor.html`, `scripts/editor-server.ts`

---

## 背景・目的

AI音声文字起こし（Whisper等）が生成するタイムスタンプはズレが生じやすく、
現状は `start`/`end` を数値で手修正するしかない。
初心者には数値入力での調整は難しいため、
音声波形を見ながらテロップをドラッグで調整できるUIを追加する。

## 機能仕様

### 波形表示
- WaveSurfer.js v7（CDN）を使用
- 音声の読み込み優先順位：
  1. `input/audio/` フォルダを自動スキャン → 見つかれば自動ロード
  2. 見つからなければ「🎵 音声を開く」ボタンを表示してユーザーが選択

### テロップブロックの操作

| 操作 | 動作 |
|------|------|
| ブロックを左右にドラッグ | start/end を同時移動 |
| ブロックの右端をドラッグ | end を調整 |
| ブロックの左端をドラッグ | start を調整 |
| ブロックをクリック | 対応するテーブル行をハイライト＆スクロール |
| 波形をクリック | その位置から音声再生 |

### ブロックの見た目
- テロップ文字をブロック内に表示（長い場合は省略）
- 選択中のブロック：色が変わる
- 重なり検出：重なっているブロックを赤くハイライト（警告）

### 一括シフト
- 「全体を -1s / +1s」ボタン：全テロップを一括移動
- チェックボックスで選択 → 「選択を -0.5s / +0.5s」ボタン：選択行のみ移動

### 保存
- ブロックをドラッグした瞬間にメモリ上のデータを更新
- 「⚡️ 保存して更新」押下時のみ transcript.json に書き込み（既存フローを流用）

## 技術仕様

### 使用ライブラリ
- `WaveSurfer.js v7`（CDN） — 波形表示・再生
- `WaveSurfer Regions Plugin`（CDN） — ドラッグ可能ブロック
- 追加インストール不要

### サーバー変更（editor-server.ts）

追加するAPIエンドポイント：

```
GET /api/audio-files   → input/audio/ の .mp3 一覧をJSON配列で返す
GET /audio/:file       → 音声ファイルをストリーム配信
```

既存の `/api/import` はそのまま流用。

### ファイル変更範囲

| ファイル | 変更内容 |
|---------|---------|
| `subtitle-editor.html` | 波形UIセクションを追加、テーブルとの双方向同期ロジックを追加 |
| `scripts/editor-server.ts` | `/api/audio-files` と `/audio/:file` エンドポイントを追加 |

---

## データフロー

```
音声ファイル
  ↓ /audio/:file（ストリーム）
WaveSurfer.js（波形描画）
  ↓ Regions Plugin
テロップブロック（ドラッグ）
  ↓ region-updated イベント
メモリ上のsegmentsデータ更新
  ↓ テーブルのstart/end入力欄に反映
「⚡️ 保存して更新」
  ↓ /api/import
transcript.json 更新
```

---

## 対象外（スコープ外）

- 波形の拡大縮小（ズーム）：初期スコープ外。後のバージョンで追加可
- テロップの新規追加を波形上で行う機能：テーブルの「＋追加」を引き続き使用
- Remotionとのリアルタイム同期：既存通り手動リロード
